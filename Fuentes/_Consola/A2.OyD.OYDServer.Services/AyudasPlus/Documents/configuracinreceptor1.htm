Configuración receptores

El objetivo de este maestro es realizar las configuraciones que se pueden realizar por receptor a nivel de tipo negocio, tipo producto, parámetros, certificaciones, conceptos y clientes autorizados.

Cuando se da clic en el botón de Nuevo o Editar. Aparece la siguiente pantalla.

Esta pantalla se compone de los siguientes campos.

•      Código receptor: código único del receptor.

•      Nombre receptor: Nombre del receptor.

•      Tipo orden: Tipo de orden (indirecta o directa) a la cual se tiene permiso.

•      Extensión defecto: Numero de la extensión del receptor.

•      Enruta Bus: Este campo permite marcar si las órdenes que realizan se envían por SAE.

•      Tipo Orden defecto: Este es el valor por defecto del tipo de orden cuando se ingresa una orden de acciones, títulos de renta fija, simultánea, Repos o TTV.

•      Porcentaje de patrimonio técnico: Este porcentaje corresponde a la participación del patrimonio técnico de la firma por receptor. Cuando el valor esta en cero no aplica la validación, y el máximo valor que puede tomar es el 100%. Este ítem se debe configurar para que aplique la regla “patrimonio técnico receptor”.

Y se divide en seis pestañas o lengüetas

1.   Tipo Negocio

Este tipo negocio corresponde al tipo de negocio en la orden, que puede ser acciones, Renta fija, repo, simultanea o TTV.

o  Prioridad. Define el orden de visualización de lista de tipo de negocio. Cuando se coloca el valor de cero este ítem se muestra por defecto. Si más de un ítem tiene el valor de cero selecciona el primer ítem para mostrarlo por defecto.

o  Valor Max negociación. Este valor se utiliza como límite de negociación por orden para el tipo de negocio seleccionado. Es importante anotar que cuando se deja el valor en cero no se limita la orden.

o  Porcentaje comisión. Este porcentaje sugerido para el tipo de negocio cuando el receptor este ingresando la orden.

o  Valor comisión. Este valor sugerido para el tipo de negocio cuando el receptor este ingresando la orden.

2.   Tipos producto.

o  Tipo producto. Esta es la clasificación de clientes definida por la firma comisionista.

o  Prioridad. Define el orden de visualización de lista de tipo producto. Cuando se coloca el valor de cero este ítem se muestra por defecto. Si más de un ítem tiene el valor de cero selecciona el primer ítem para mostrarlo por defecto.

3.   Parámetros.

o  Lista de selección “Tópico o temas”.

o  Seleccione. Marca la opción si está disponible o no.

o  Código. Código único de la lista.

o  Tópico. Nombre del tópico.

o  Valor. Valor de retorno de la lista.

o  Descripción. Descripción de la lista.

o  Prioridad. Define el orden de visualización de los tópicos. Cuando se coloca el valor de cero este ítem se muestra por defecto. Si más de un ítem tiene el valor de cero selecciona el primer ítem para mostrarlo por defecto.

Ejemplos:

•      Cuando no se selecciona ninguna de las “Formas de Pago”.

Al momento de ingresar la orden, cuando se selecciona Forma de pago aparecen todos los datos de la lista.

•      Cuando se selecciona “Forma de Pago” aparecen las opciones que tienen se tienen configuradas para la firma.

Al momento de ingresar la orden, cuando se selecciona forma de pago aparecen solo las dos formas de pago que se marcaron y en el orden de prioridad asignado. En caso que se coloque una prioridad con cero el dato se muestra por defecto.

•      Cuando se selecciona “Forma de Pago” aparecen las opciones que tienen se tienen configuradas para la firma

Al momento de ingresar la orden, cuando se selecciona forma de pago aparecen solo las dos formas de pago que se marcaron y la forma de pago “1-cheque” se visualiza por defecto por tener prioridad cero.

4.   Certificaciones

o  Certificación. Lista de certificaciones.

o  Fecha inicial. Fecha de inicio de vigencia.

o  Fecha final. Fecha de fin de vigencia.

Es importante anotar que si no se tienen configuradas las certificaciones por receptor así esta configurada la regla “CERTIFICACIONESRECEPTOR” no aplica la validación.

5.   Conceptos

o  Concepto. Lista de conceptos de tesorería. Estos conceptos solo aplican a los comerciales que ingresan órdenes de tesorería.

o  Prioridad. Define el orden de visualización de los conceptos de tesorería. Cuando se coloca el valor de cero este ítem se muestra por defecto. Si más de un ítem tiene el valor de cero selecciona el primer ítem para mostrarlo por defecto.

6.   Clientes Autorizados

En esta pestaña se permite autorizar al receptor A con clientes autorizados de otro receptor B. Ejemplo cuando un receptor se va a vacaciones.

o  Código Receptor. Código del receptor.

o  Nombre Receptor. Nombre del receptor.

o  Código OYD. Cuando aplica muestra el código de cliente en OYD.

o  Nombre cliente. Nombre del cliente, cuando aparece todos es que aplica para todos los clientes.

o  Fecha inicial. Fecha de inicio de vigencia.

o  Fecha final. Fecha de fin de vigencia.

Para ingresar un nuevo registro se debe dar clic en el botón nuevo.

Luego se selecciona a buscar el receptor B
