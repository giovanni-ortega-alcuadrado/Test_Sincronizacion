Operaciones por Otras Firmas (Ellos Operan)

Este tipo de operación es una operación que se realiza en BVC pero a través de otra firma comisionista donde la posición propia es cliente, esta operación NO queda registrada  en OyD en el módulo de Bolsa normal, queda registrada en el módulo denominado “Operaciones por otras firmas”. Se debe adicionar a los tipos de negocio:

•      Otras firmas acciones

•      Otras firmas renta fija

 

12.  El usuario registrara la Orden tal y como hace con las órdenes para OyD de Bolsa pero estas quedaran matriculadas en el módulo de Órdenes de Otras firmas (OF).

Si se selecciona el tipo de negocio “Otras firmas acciones” se debe mostrar los siguiente campos.

Si es Tipo orden “Indirecta”  y el Tipo operación “Compra”   aparece la pestaña de “Saldo” y “Bolsa”  como funciona actualmente.

Si es Tipo orden “Directa”  y tipo de operación “Compra”  aparece la pestaña de “Saldo” y “Bolsa” como funciona actualmente.

Si es Tipo orden “Indirecta”  y el Tipo operación “Venta”  aparece la pestaña de “Portafolio”; como funciona actualmente.

Si es Tipo orden “Directa”  y tipo de operación “Venta”  aparece la pestaña de “Portafolio” y “Bolsa” como funciona actualmente.

Si se selecciona el tipo de negocio “Otras firmas Renta fija” se debe mostrar los siguientes campos.

Las pestaña de “Saldo”, “portafolio” y “Bolsa” funcionan de la misma forma como están actualmente y se detalla en el punto anterior de “otras firmas acciones”.

 

Validaciones al momento de grabar.

 

Para la entrega del desarrollo de AV-Ticket, las órdenes de OTRAS FIRMAS tendrán las siguientes validaciones:

 

•      Saldo en dinero para compras. Cuando el cliente realiza una compra se valida que tenga el saldo necesario para cubrir la compra.

•      Portafolio en títulos para ventas. Antes de vender se debe validar que el cliente los tenga los títulos a vender en el portafolio.

•      Configuración. Que el receptor tenga el tipo de  producto, que tenga las certificaciones, las configuraciones establecidas para el receptor y el tipo de negocio.

•      Cupos por receptor y por cliente.

•      Clientes relacionados.

•      Control de horarios.  

Si no cumple con alguna de las reglas mencionadas en punto anteriores, y la regla está configurada para solicitar autorización, se debe ir para autorizaciones al nivel de atribución configurado.

13.  Las órdenes de OTRAS FRIMAS deben caer a la pantalla del seteador, pero se debe identificar que la orden está clasificada como OTRAS FIRMAS.

14.  Operativamente el área de operaciones matriculara estas papeletas en el módulo de OF para que sigan afectando la operatividad normal de la firma.

15.  Estas operaciones afectan la posición del comercial en títulos de la posición propia y se visualizará en la siguiente pantalla.
