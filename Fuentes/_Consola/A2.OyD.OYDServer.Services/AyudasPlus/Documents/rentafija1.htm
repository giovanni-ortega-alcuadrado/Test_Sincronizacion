Renta fija

Si se selecciona tipo negocio = Renta fija, con tipo operación compra o venta.

Encabezado de la orden

Luego que seleccionan los datos y se selecciona tipo de negocio “Renta fija” abre la orden con los siguientes campos por defecto.

o  Mercado. Este campo corresponde al mercado donde se ejecuta la orden y por defecto trae el valor de “secundario”. Este campo es obligatorio.

o  Tipo limite. Este campo determina el precio, este dato puede ser límite, mercado o mercado por los mejor. Este campo es obligatorio.

o  Ejecución. Este campo determina el criterio de calce y puede ser:

Duración

Disponible para Tipo negocio

Ninguna

Acciones, Renta fija

Campo que se deben digitar para el ingreso de la orden

o  Clasificación. Este campo determina si la orden es “Ordinaria”, si se selecciona un dato diferente a ordinaria la orden se considera extraordinario. Y Este campo es obligatorio.

o  Condiciones negociación. Este campo es obligatorio.

o  Forma pago. Este campo es obligatorio y corresponde a las formas de pago. Ejemplo cheques, efectivo, pago red banco.

o  Tipo inversión. Este campo corresponde a l tipo de inversión y es obligatorio cuando se selecciona tipo de producto “posición propia”.

o  Duración. Este campo determina la duración de la orden, dependiendo de la selección de este campo se determina la fecha de vigencia. Y datos que trae la selección pueden ser:

Duración

Disponible para Tipo negocio

Parametrización

Hasta Cancelación

Acciones, Renta fija

Los días de cancelación se obtienen de la configuración parámetros con el nombre DIASCANCELACIONORDEN, es configurable por el administrador.

Hasta fecha

Acciones, Renta fija

Cuando se selecciona esta opción habilita el campo fecha para escoger una fecha determinada.

o  Cliente. Este campo corresponde al cliente para el que se realiza la orden. Este campo es obligatorio.

Ejemplo del buscador de cliente.

o  Si de da clic en el botón de la lupa sin digitar datos se trae todos los clientes disponibles del receptor con el tipo de producto seleccionado.

Si se desea realizar una búsqueda específica se ingresa el  código OYD del cliente completo o parte, numero de documento de cliente o parte, nombre completo o parte.

Una vez seleccionado presenta los datos del cliente de la siguiente forma:

o  Ordenante. Si el cliente tiene un solo ordenante lo muestra de lo contrario se debe escoger de la lista. Este campo es obligatorio.

o  Cuenta deposito. Si el cliente tiene una sola cuenta depósito la muestra de lo contrario se debe seleccionar de la lista. Este campo es obligatorio.

o  Especie. Este campo es obligatorio. Una vez se selecciona la especie, se pueden seleccionar los ISINES configurados para la especie.  Estas especies son las de renta fija y exige que obligatoriamente se diligencie el ISIN, Fecha emisión, Fecha vencimiento, Tasa facial y modalidad.

Pestaña de distribución de comisiones

Cuando se está ingresando la orden se sugiere el 100% con el receptor de la orden. Es obligatorio tener el porcentaje de distribución igual al 100 por ciento.

Ejemplo:

Se ingresa un orden con el receptor EGL, tipo orden Indirecta, tipo producto 1 Portafolio de Terceros, Tipo negocio Renta fija y Tipo de operación Compra, clasificación Ordinaria, mercado Secundario, Tipo límite Con límite, Condiciones negociación A plazo, Forma pago 1-Cheque, Ejecución Ninguna, Duración Hasta fecha, Cliente 51933, especie CDTBBOS0V, Nominal 1000, precio 158, Tasa mercado 5, Tasa cliente 6.

Presenta las reglas que están con acción de regla de tipo advertencia.

Se da clic en grabar.

Se consulta la orden en OYD en el menú Bolsa >> Ordenes >> Renta Fija.

Y se edita la orden para ver el detalle.

Si se da clic en “Editar”, presenta la validación que solo puede ser modificada en OyD plus.

Si se da clic en “Borrar”, presenta la validación que solo puede ser modificada en OyD plus.

Y continúa con el flujo normal de OyD, ejemplo de como se ve en “Liquidaciones”.
