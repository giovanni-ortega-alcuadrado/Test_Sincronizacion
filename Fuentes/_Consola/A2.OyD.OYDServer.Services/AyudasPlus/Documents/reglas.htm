•      Código Regla: El código de la regla es un valor único que identifica la regla.

•      Descripción Regla: La descripción de regla es el nombre largo para identificar la regla de manera funcional o de negocio.

•      Modulo: Identifica el modulo pare el que aplica la regla. Los módulos actuales para OYD plus son Ordenes, Ordenes de tesorería.

•      Nombre Corto: La descripción corta de regla para identificar la regla de manera funcional o de negocio.

•      Prioridad: Esta es la prioridad para aplicar la regla.

•      Acción Regla: Esta una lista de selección de acciones que se pueden aplicar a las regla.

o  NO validar: Este ítem permite marcar cuando la regla no se valida.

o  No continua (Detiene el proceso): Este ítem permite marcar que si la regla en caso de incumplirla detiene el ingreso de la orden.

o  Genera Advertencia: Este ítem permite marcar que si la regla en caso de incumplirla continúa y muestra la advertencia.

o  Requiere Justificación: Este ítem permite marcar cuando la regla requiere justificación.

o  Requiere Aprobación: Este ítem permite marcar cuando la regla requiere aprobación.

•      Causas Justificación: En este texto se permite adicionar la lista de justificaciones para la regla; cuando se configure que requiere justificación.

2. Regla de Patrimonio Técnico: esta regla valida en el ingreso y modificación de ordenes repo vigentes por parte del cliente que no se pasen del patrimonio técnico de la firma. Y se basa en el valor configurado en el parámetro “PATRIMONIOTECNICO”

•      La validación se hará a nivel de Documento de Identidad del cliente para el total de la firma no podrán haber valores diferentes de patrimonio técnico. Es decir se sumaran las operaciones de todos los códigos asociados al documento de identidad del cliente que se está matriculando en la orden.

•      La validación solo se hará sobre las operaciones REPO de venta Inicial.

•      Es importante anotar que si este parámetro está en CERO o NO EXISTE quiere decir que la validación no se realizara. Esto se hace para que solo aplique para la firma.

 

13. Regla Patrimonio técnico receptor: esta regla valida en el ingreso y modificación de ordenes repo vigentes por parte del receptor que no se pasen del porcentaje del patrimonio configurado por receptor. Este porcentaje se basa en el valor configurado en el parámetro “PATRIMONIOTECNICO”

•      La validación solo se hará sobre las operaciones REPO de venta Inicial. Y se realiza solo si el parámetro “PATRIMONIOTECNICO” es diferente a CERO.

•      Es importante anotar que si este porcentaje está en CERO no se realizara la validación por receptor.

 

14. Regla Patrimonio técnico sucursal: esta regla valida en el ingreso y modificación de ordenes repo vigentes por sucursal que no se pasen del porcentaje del patrimonio configurado por sucursal. Este porcentaje se basa en el valor configurado en el parámetro “PATRIMONIOTECNICO”

•      La validación solo se hará sobre las operaciones REPO de venta Inicial. Y se realiza solo si el parámetro “PATRIMONIOTECNICO” es diferente a CERO.

•      Es importante anotar que si este porcentaje está en CERO no se realizará la validación por sucursal.

 

15. Regla Patrimonio técnico cliente: esta regla valida en el ingreso y modificación de ordenes repo vigentes por sucursal que no se pasen del porcentaje del patrimonio configurado por sucursal. Este porcentaje se basa en el valor configurado en el parámetro “PATRIMONIOTECNICO”

•      La validación solo se hará sobre las operaciones REPO de venta Inicial. Y se realiza solo si el parámetro “PATRIMONIOTECNICO” es diferente a CERO.

•      Es importante anotar que si este porcentaje está en CERO no se realizará la validación por cliente.
