Cómo aplica la regla “PATRIMONIOTECNICORECEPTOR”

 

Esta regla permite validar el patrimonio técnico del receptor. La configuración de la regla en este caso está activa porque la acción regla es igual a “No continua (detiene el proceso)”.

Se tiene el receptor EGL con un porcentaje de patrimonio 13%.

 

Parámetros que aplica:

•      'BOLSA_VALIDARLIMITES_PORTIPOESPECIE'. Define el tipo de negocio que se tendra en cuenta en los sps de validacion de limites para el decreto 2878. Ejemplo RTSP. Significa que valida Repos, TTV, Simultáneas y ordenes A Plazo.

•      'BOLSA_VALIDARLIMITES_PORTIPODENEGOCIO'. Define el tipo de especie que se tendra en cuenta en los sps de validacion de limites para el decreto 2878. Ejemplo AC. Significa que toma especies de Acciones o Crediticias (Renta fija).

•      'BOLSA_VALIDARLIMITES_SUMARPORTIPODENEGOCIO'. Define el tipo de negocio que se tendra en cuenta en el total para los sps de validacion de limites para el decreto 2878. Ejemplo RTSP. Significa que valida Repos, TTV, Simultáneas y ordenes A Plazo.

•      'PRODUCTO_CUENTADEMARGEN. Define el retorno del tipo de producto cuenta de margen. Ejemplo C.

•      'PATRIMONIOTECNICO' En este parametro de define el patrimonio tecnico de la firma.

 

Al momento de ingresar una orden se valida cuando se dan las siguientes condiciones:

•      Se evalúa si es Repo Pasiva. Es tipo operación igual a “venta”, el tipo de negocio es “Repo” y se encuentra configurado el tipo de negocio.

•      Se evalúa si es TTV Pasiva. Es tipo operación igual a “venta”, el tipo de negocio es “TTV” y se encuentra configurado el tipo de negocio.

•      Se evalúa si las orden es a Plazo. Condición de negociación es igual a “A plazo” y se encuentra configurada la condición de negocio.

•      Se evalúa si es Simultánea y se encuentra configurado el tipo de negocio.
