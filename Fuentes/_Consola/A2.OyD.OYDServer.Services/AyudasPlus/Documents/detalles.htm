Detalles

1.   Pestaña de cheques

Si se selecciona la pestaña de “cheques” se debe desplegar las siguientes columnas:

•      Tipo cliente. Puede ser cliente o tercero. Si se selecciona cliente automáticamente trae los datos de nombre beneficiario, tipo documento y número documento. Si se selecciona tercero se debe digitar los datos de nombre beneficiario, tipo documento y número documento.

Es importante anotar que esta información se valida con las listas restrictivas (personas inhábiles) al guardar la orden.

•      Concepto: se seleccionable de las lista de Conceptos Tesorería. Esta lista de conceptos es configurable en el maestro Configuración receptores, en la pestaña de Conceptos. Si no se tiene ningún concepto matriculado se trae todos los conceptos.

•      Detalle Concepto. Es un texto que el usuario puede digitar adicional. Esto se graba concatenado en la generación del Egreso.

 

Se debe tener en cuenta que es obligatorio como mínimo tener Concepto o Detalle Concepto para grabar el ítem de cartera colectiva.

 

•      Tipo  cheque: Puede ser cheque o cheque gerencia.

•      Tipo cruce: Si se seleccionó cheque se permite escoger el tipo cruce. Estos datos se traen de la lista.

•      Valor a generar: Este valor se digita.

•      Tipo GMF. Puede ser  encima, por debajo.

•      Valor GMF. Si es un tercero se calcula. Si es el mismo cliente. Se deja en cero. Este campo no es editable (si no está configurado).

•      Valor Neto es igual a valor con GMF.

Importación de archivo para cheques

Se adjunta la definición de la carga de archivos.

Cuando el usuario selecciona la opción de carga de archivo plano de cheques, debe traer los campos adjuntos en el archivo y tener la opción el usuario que carga de definir los campos por defecto que no vienen en el plano así:

Cuando carga el archivo el sistema identifica si el cheque es a un Tercero o a un cliente  esto con el documento de identidad asociado al cliente del plano contra el documento del cliente de la orden. 

Si es TERCERO en la pantalla de carga tendrá la opción de escoger un “por defecto” para todos los registros cargados, si el GMF es por encima o por debajo y a todos los clientes que sean terceros le cobra con la definición que tenga este campo por “defecto en la pantalla”. Y se realiza el cálculo del GMF por encima o por debajo de acuerdo a la lógica conocida y el neto del pago de acuerdo a esta definición.

El tipo de Cheque, el tipo de cruce estarán en la pantalla de carga con el fin de que al momento de cargar el archivo TODOS los pagos que se carguen tengan estos dos campos por defecto.

5.   Bloqueo o liberar recursos

Si se selecciona la  pestaña de “Bloqueo Recursos” que tiene como objetivo permitir hacer una orden para bloquear el saldo de cliente. Se ingresa un registro en la opción de Bloquear Saldos y se genera el documento de Tesorería con este bloqueo.

 

Esta pestaña contiene los mismos datos que actualmente maneja OYD.

•      Motivo bloqueo Saldo. Este es un campo de lista de datos (combo) con tópico igual a “MOTIVOBLOQUEOSALDO”

•      Valor bloqueado. Valor numérico y digitable.

•      Naturaleza. Este es un campo de lista de datos (combo). Crédito o débito. Investigar cómo queda para bloquear saldo.

•      Detalle de bloqueo. Este es un campo de observación.

 

En la pantalla del tesorero se debe agregar en formas de pago “Bloqueo recursos”, en el momento de consultar se presentan los detalles que se ingresen con este ítem.

Al momento de generar documento de debe generar la nota de bloqueo de saldo por garantía.
