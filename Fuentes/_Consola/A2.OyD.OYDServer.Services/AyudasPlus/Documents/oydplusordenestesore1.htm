La orden de giro (Instrucción de pago emitida por la comisionista a favor de un cliente o tercero con una forma de pago y en una fecha determinada) se compone de un encabezado y sus detalles o formas de pago.

Encabezado

1.   Carta de autorización. Este campo permite adjuntar un archivo con la carta de autorización del cliente para la orden de giro.

2.   Receptor. Presenta la lista de los receptores matriculados al usuario que accede a la aplicación.

3.   Tipo de producto. Para este caso  se selecciona el tipo de producto “Fondos”

4.   Fecha de orden. Determina la fecha de ejecución de la orden.

5.   Cliente. En esta lista se encuentran los clientes a los cuales tiene permiso el receptor seleccionado.

6.   Ordenante. En esta lista se encuentran los ordenantes matriculados para el cliente seleccionado.

7.   Cartera colectiva. Lista de carteras colectivas o fondos que posee el cliente seleccionado.

8.   Nro. Encargo. Lista de números de encargo según el cliente y la cartera colectiva seleccionada.

9.   Tipo acción: Esta es una lista en la cual se puede seleccionar si la orden de fondos es de cancelaciones o retiros.

10.  Saldo actual. Composición del saldo dependiendo del cliente seleccionado. Para este caso como se tiene seleccionado el tipo producto “Fondos”, el saldo se compone de saldo actual, saldo neto, Lista de fondos y encargo, órdenes de giro, ordenes de recibo (adición) y ordenes de recibo (constitución). Se debe tener en cuenta que para presentar el Saldo se debe seleccionar el receptor, tipo producto “tesorería fondos”, cliente y cratera colectiva.

Formas de pagos disponibles

•      Cheques. Permite realizar un pago a un tipo cliente igual a Cliente o tercero, con un concepto, con tipo cheque que puede ser cheque o cheque gerencia y con un valor.

•      Trasferencias. Permite realizar pago a una cuenta registrada o  una no registrada, con un concepto y con un valor.

•      OyD. Cuando se realiza un retiro o cancelación del fondo y se lleva a OYD. Permite realizar pago a un tipo cliente igual a Cliente o tercero, con un concepto, y con un valor.

•      Operaciones Especiales. Permite realizar pago con tipo (mercado secundario, endosos, cesiones, operaciones especiales, pignoraciones), si se selecciona el tipo operaciones especiales entonces se activa la lista operación especial  (aporte en especie, donación), código OyD comprador, código OyD vendedor, proviene dinero, observación y valor.
