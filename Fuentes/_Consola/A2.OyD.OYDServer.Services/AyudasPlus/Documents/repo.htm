Si se selecciona tipo negocio = Repo, con tipo operación compra (activa) o venta (pasiva).

•      Luego que seleccionan los datos y se selecciona tipo de negocio “Renta fija” abre la orden con los siguientes campos por defecto.

o  Clasificación. Este campo por defecto toma el valor de “repo”, Que es la única clasificación disponible Y Este campo es obligatorio.

o  Mercado. Este campo corresponde al mercado donde se ejecuta la orden y por defecto trae el valor de “Repo”, que es el único mercado para este tipo negocio. Este campo es obligatorio.

o  Tipo límite. Este campo determina el precio, este dato puede ser límite, mercado o mercado por los mejor, para el caso de los repos se trae por defecto “Mercado por lo mejor”. Este campo es obligatorio.

o  Ejecución. Este campo determina el criterio de calce y puede ser: Es igual a “Ninguna” y no puede ser modificado.

Formulas

Si la operación es venta

Precio = Valor Acción * (1 -(Castigo/100))

Valor Futuro Repo = Valor Captación Giro * (1 + (Tasa Registro / 100) * (Días Repo / 360))

Cantidad = Valor Futuro Repo / Precio + 1

Valor Orden = (Precio * Cantidad) / (1 + (1 / 100) * (Días Repo / 360))

Si la operación es compra

Valor Futuro Repo = Valor Captación Giro * (1 + (Días Repo / 360))

Cantidad = Valor Futuro Repo / Precio + 1

Valor Orden = (Precio * Cantidad) / (1 + (1 / 100) * (Días Repo / 360))

•      Distribución de comisiones

 

Cuando se está ingresando la orden se sugiere el 100% con el receptor de la orden. Es obligatorio tener el porcentaje de distribución igual al 100 por ciento.
