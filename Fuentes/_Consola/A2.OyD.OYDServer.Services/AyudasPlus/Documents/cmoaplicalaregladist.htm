Cómo aplica la regla “DISTRIBUCIONPORTAFOLIOCLIENTE”

Esta regla permite validar la distribución del portafolio y cupo x tipo de negocio para el cliente. La configuración de la regla en este caso está activa porque la acción regla es igual a “Generar advertencia”.

Pasos para verificar los datos para la regla

1.   Informe general de custodias ampliado al día anterior valorizado para el cliente especifico.

2.   Elimino registros bloqueados en Repo activo y repo pasivo

3.   Lo queda es  lo que tiene el cliente en portafolio para acciones, RF tanto en nominales como en valorización.

Estos puntos pueden ser empaquetados o calculados en la mañana al día anterior

4.   Consulto las liquidaciones por cumplir con corte en el día; es decir F liquidación <= hoy y F cumplimiento >= hoy. Esto se obtiene con el informe de liquidaciones por cumplir

5.   Separar acciones, renta fija, simultanea, repos, ttv. Estos contiene Cantidad/nominal > total liquidación (valoración)

6.   Se agrega las ordenes pendientes de acciones, renta fija, simultanea, repos, ttv (cantidad y valor total)

Ejemplo: Se configura para todos los tipos de producto “todos”, perfil de riesgo”2. Moderado”, con clientes “todos”:

•      Acciones           50%

•      Renta fija           50%

Y se tiene un cliente “51957” con perfil de riesgo “2- Moderado”.

Al momento de ingresar una orden para el cliente “51957” como el cliente tiene perfil de riesgo “2-Moderado” y es una orden de tipo operación “venta”

Si el cliente no tiene portafolio (custodias) no permite ingresar la orden. Para permitir ingresar la orden se tendría que desactivar la regla; es decir colocar el campo acción regla igual a “No validar”
