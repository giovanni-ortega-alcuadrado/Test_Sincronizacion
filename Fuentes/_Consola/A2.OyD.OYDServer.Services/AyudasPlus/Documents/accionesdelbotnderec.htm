Acciones del botón derecho del mouse

En la pestaña de Pendientes

•      Mostrar en visor. Cuando se da clic en esta opción se abre el visor para la orden seleccionada. . Con el visor se tiene la opción de lanzar la orden o rechazarla según el caso.

•      Marcar como lanzada. Cuándo se selecciona esta opción, el programa  realiza el proceso de marcar la orden como lanzada y pasa a la pestaña de procesadas

•      Lanzar SAE. Este botón debe ser configurable por tipo de negocio; actualmente solo debe habilitarse para acciones. Cuando se da clic en esta opción, se realiza el proceso de lanzar la orden por SAE. Adicionalmente se debe de tener en cuenta que cuando se va enviar documentos al bus de integración. Los usuario con los cuales se está trabajando deben de tener configurado un Consecutivo en el maestro de consecutivos usuario “ENRUTARORDEN”.

 

Consecutivo por usuario

•      Rechazar. Una vez se selecciona  esta opción el programa muestra una lista de causas  de rechazo y una casilla de texto para colocar la observación pertinentes a este.

En la pestaña de Procesadas:

•      Mostrar en visor. Cuando se presiona este botón, se abre el visor con  la orden seleccionada y muestra la información correspondiente, solo se muestra en modo consulta ya que las opciones las muestra deshabilitadas

•      Asociar Liquidaciones. Con esta opción el programa permite relacionar las órdenes con las diferentes operaciones de la Bolsa de Valores de Colombia, se puede seleccionar  una o varias liquidaciones notificadas. Estas quedan en Liquidaciones probables, en ese caso el ejecutor verifica la información de BVC y de la orden de OYD. Se debe tener en cuenta que el operador debe tener el mismo código de la operaciones y estar matriculado con el sistema registrado en SAE. Para verificar estos datos se consulta por el maestro de receptores.

El código del sistema SAE se matricula por el maestro de configuración con nombre “CODIGOSISTEMASAE”.

Ejemplo se selecciona la orden 2013058877 de CEMARGOS con cantidad de 7000 acciones.

Cuando se hace clic en asociar liquidaciones.

Trae dos operaciones cada una con 20000 acciones disponibles. Al seleccionar la primera con folio 1934. Se debe asignar a la orden una liquidación probable por 7000 con esa operación.

 Y debe quedar un saldo de 13000 acciones disponibles para la operación con folio número 1934.

Es decir que en el momento solo permite generar ordenes con la cantidad o nominal  menor igual al monto de la operación.

Esta validación aplica para todos los tiempos de negocio lo que se debe tener en cuenta es que si hablamos de acciones se maneja cantidad, si hablamos de títulos de renta fija se maneja nominal.

Entonces siguiente con el ejemplo, se pueden tener varias órdenes asignadas a una operación con diferentes parciales.
