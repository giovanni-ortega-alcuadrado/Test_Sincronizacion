Acciones

Si se selecciona tipo negocio = Acciones, con tipo operación compra o venta.

Encabezado de la orden

Luego que seleccionan los datos y se selecciona tipo de negocio “Acciones” abre la orden con los siguientes campos por defecto.

•      Clasificación. Este campo por defecto toma el valor de “Ordinaria”, si se selecciona un dato diferente a ordinaria la orden se considera extraordinario. Y Este campo es obligatorio.

•      Mercado. Este campo corresponde al mercado donde se ejecuta la orden y por defecto trae el valor de “secundario”. Este campo es obligatorio.

•      Tipo límite. Este campo determina el precio, este dato puede ser límite, mercado o mercado por los mejor. Este campo es obligatorio.

•      Ejecución. Este campo es obligatorio, determina el criterio de calce y puede ser:

Duración

Definición

Disponible para Tipo negocio

Cantidad mínima

La oferta se debe calzar por lo menos una mínima cantidad, de lo contrario el sistema la retira.

Acciones

Fill or kill

Si la orden no puede ser calzada en su totalidad el sistema la retira sin generar ningún calce.

Acciones

Fill and kill

La orden se calza por cierta cantidad y el remanente es retirado automáticamente de la rueda.

 

Acciones

Ninguna

La orden no tiene ningún criterio adicional de calce.

Campo que se deben digitar para el ingreso de la orden

•      Condiciones negociación. Este campo es obligatorio.

•      Forma pago. Este campo es obligatorio y corresponde a las formas de pago. Ejemplo cheques, efectivo, pago red banco.

•      Tipo inversión. Este campo corresponde a l tipo de inversión y es obligatorio cuando se selecciona tipo de producto “posición propia”.

•      Cliente. Este campo corresponde al cliente para el que se realiza la orden. Este campo es obligatorio.

Ejemplo del buscador de cliente.

•      Si de da clic en el botón de la lupa sin digitar datos se trae todos los clientes disponibles del receptor con el tipo de producto seleccionado.

Si se desea realizar una búsqueda específica se ingresa el  código OYD del cliente completo o parte, numero de documento de cliente o parte, nombre completo o parte.

Una vez seleccionado presenta los datos del cliente de la siguiente forma:

•      Ordenante. Si el cliente tiene un solo ordenante lo muestra de lo contrario se debe escoger de la lista. Este campo es obligatorio.

•      Cuenta deposito. Si el cliente tiene una sola cuenta depósito la muestra de lo contrario se debe seleccionar de la lista. Este campo es obligatorio.

•      Especie. Este campo es obligatorio. T trae las especies configuradas para acciones.

Ejemplo:

Se ingresa un orden con el receptor EGL, tipo orden Indirecta, tipo producto 1 Portafolio de Terceros, Tipo negocio Acciones y Tipo de operación Venta, clasificación Ordinaria, mercado Secundario, Tipo límite Mercado, Condiciones negociación Contado, Forma pago 1-Cheque, Ejecución Ninguna, Duración Inmediata, Cliente 274, especie BIOMAX, cantidad 1000, precio 1300, % comisión 0.300.

La orden presenta la validación de los rangos de comisión.

Se modifica la orden para que este entre los rangos de comisión. Modificó el porcentaje de comisión a 0.500 y se da clic en grabar. Presenta las reglas que están con acción de regla de tipo advertencia.

Se da clic en grabar.

Se consulta la orden en OYD en el menú Bolsa >> Ordenes >> Acciones.

Y se edita la orden para ver el detalle.

Si se da clic en “Editar”, presenta la validación que solo puede ser modificada en OyD plus.

Si se da clic en “Borrar”, presenta la validación que solo puede ser modificada en OyD plus.

Y continúa con el flujo normal de OyD, ejemplo de como se ve en “Liquidaciones”.
