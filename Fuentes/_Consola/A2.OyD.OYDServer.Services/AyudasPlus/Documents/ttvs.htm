TTV´s

Si se selecciona tipo negocio = TTV, con tipo operación compra (activa) o venta (pasiva).

•      Luego que seleccionan los datos y se selecciona tipo de negocio “Renta fija” abre la orden con los siguientes campos por defecto.

o  Clasificación: Este campo por defecto toma el valor de “TTV Salida”, Que es la única clasificación disponible Y Este campo es obligatorio.

o  Mercado. Este campo corresponde al mercado donde se ejecuta la orden y por defecto trae el valor de “secundario”. Este campo es obligatorio.

o  Tipo límite. Este campo determina el precio, este dato puede ser límite, mercado o mercado por los mejor, para el caso de las simultaneas se trae por defecto “Mercado por lo mejor”. Este campo es obligatorio.

o  Ejecución: Este campo determina el criterio de calce y puede ser: Es igual a “Ninguna” y no puede ser modificado.

•      Con respecto a los otros campos.

o  Condiciones negociación. Este campo es obligatorio.

o  Forma pago. Este campo es obligatorio y corresponde a las formas de pago. Ejemplo cheques, efectivo, pago red banco.

o  Tipo inversión. Este campo corresponde a l tipo de inversión y es obligatorio cuando se selecciona tipo de producto “posición propia”.

o  Duración. Este campo determina la duración de la orden, dependiendo de la selección de este campo se determina la fecha de vigencia. Y datos que trae la selección pueden ser:

Cuando se selecciona esta opción habilita el campo fecha para escoger una fecha determinada.

 

o  Cliente. Este campo corresponde al cliente para el que se realiza la orden. Este campo es obligatorio.

Ejemplo del buscador de cliente.

o  Si de da clic en el botón de la lupa sin digitar datos se trae todos los clientes disponibles del receptor con el tipo de producto seleccionado.

Si se desea realizar una búsqueda específica se ingresa el  código OYD del cliente completo o parte, numero de documento de cliente o parte, nombre completo o parte.

Una vez seleccionado presenta los datos del cliente de la siguiente forma:

o  Ordenante. Si el cliente tiene un solo ordenante lo muestra de lo contrario se debe escoger de la lista. Este campo es obligatorio.

o  Cuenta deposito. Si el cliente tiene una sola cuenta depósito la muestra de lo contrario se debe seleccionar de la lista. Este campo es obligatorio.

o  Especie. Este campo es obligatorio. Estas especies son las de acciones y renta fija.

Pestaña de datos del negocio

Cuando es venta (originador)

Cuando es una compra (receptor)

Escenario de cálculo para compra o venta. Se digita cantidad, precio. Calcula valor giro, valor futuro y neto orden.

Compra (receptor)

El cálculo que se hace es cantidad por precio. La tasa registro y tasa cliente no intervienen en el cálculo.

Venta (originador)

El cálculo que se hace es cantidad por precio. La tasa registro y tasa cliente no intervienen en el cálculo.

Pestaña Distribución de comisiones

Cuando se está ingresando la orden se sugiere el 100% con el receptor de la orden. Es obligatorio tener el porcentaje de distribución igual al 100 por ciento.

Pestaña Datos del Leo

Los datos de Leo que son obligatorios son el canal recepción, usuario operador y medio verificable.

Ejemplo:

Se ingresa un orden con el receptor 111, tipo orden Indirecta, tipo producto 1 Portafolio de Terceros, Tipo negocio TTV y Tipo de operación Activa, clasificación TTV de Salida, mercado Secundario, Tipo límite Con limite, Condiciones negociación A plazo, Forma pago 1-Cheque, Ejecución Ninguna, Duración Hasta Fecha, Cliente 51960, especie ISAGEN, cantidad 12000, precio 2870, Tasa mercado 4, Tasa cliente 5.6.

Se da clic en grabar.

Se consulta la orden en OYD en el menú Bolsa >> Ordenes >> Renta Fija. Como se muestra en la imagen como corresponde a un Repo crea la orden de salida 2014001624 con tipo operación compra y la orden de regreso 2014001625 con tipo operación venta.

Y se edita la orden para ver el detalle de la orden de salida.

Y se edita la orden para ver el detalle de la orden de regreso.

Si se da clic en “Editar”, presenta la validación que solo puede ser modificada en OyD plus. Y si se da clic en “Borrar”, presenta la validación que solo puede ser modificada en OyD plus. Y se continúa con el flujo normal de OyD.
