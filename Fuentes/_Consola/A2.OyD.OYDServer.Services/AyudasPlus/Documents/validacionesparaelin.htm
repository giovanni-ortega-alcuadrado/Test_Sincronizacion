Validaciones para el ingreso a la pantalla órdenes.

1.   Configuración de tipo de negocio. Mensaje “Señor usuario, el receptor seleccionado no tiene tipos de negocio configurados”.

Este maestro es definido por la aplicación y no está disponible para los usuarios.

2.   Configuración de usuarios receptor. Mensaje “No hay ningún receptor configurado para el usuario”.

Revisar la ortografía del mensaje. La palabra ningún lleva tilde.

En estos momentos existe un maestro en OYD en el Menú: Maestros à Estructura comercial à Filtro Usuarios Receptor / Sucursal. Pero cuando se revisa el usuario de la aplicación se trae de las utilidades sin dominio (revisar este tema porque se le quita el dominio y no lo muestra en la orden) y adicionalmente hace falta el campo prioridad.

3.   Configuración de parámetros adicionales receptor. Mensaje “Señor usuario, el receptor no tiene configurado datos en el maestro de configuración adicionales receptor.”

En este maestro se configura el tipo orden disponible, la extensión telefónica y el valor por defecto del tipo de orden.

4.   Configuración de Tipos productos receptor

5.   Configuración de clientes asociados.

Primero Clientes receptores, segundo que los clientes tengan configurado el tipo producto y tercero que el tipo de negocio y tipo están asociados a un cliente o perfil riesgo.

6.   Configuraciones de tipo negocio, regla restrictiva con respecto a los máximos. El mensaje “El cliente solo tiene permitido negociar 0 pesos para el tipo de negocio” está asociado con el maestro Tipo Negocio por Tipo Producto y el mensaje “La especie solo tiene permitido negociar 0 peos para el tipo negocio acciones” está asociado con el maestro Tipo Negocio por Especie

7.   Configuración de certificaciones.

Certificaciones: Se matriculo la certificación de acciones con fecha inicio 01-Ene-2012 y fecha hasta 18-Nov-2012.
