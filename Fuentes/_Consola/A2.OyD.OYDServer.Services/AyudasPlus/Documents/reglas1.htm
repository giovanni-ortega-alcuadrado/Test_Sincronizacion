Reglas

En esta pantalla están definidas las diferentes reglas disponibles para el manejo de la aplicación.

Ilustración 3 Página 1 Reglas

Cuando se da clic en el botón Editar. Aparece la siguiente pantalla.

En esta se tienen dos ítems que no son modificables porque determinan la identificación de la regla.

•      Código Regla: El código de la regla es un valor único que identifica la regla.

•      Modulo: Identifica el modulo pare el que aplica la regla. Los módulos actuales para OYD plus son Ordenes, Ordenes de tesorería.

Los demás ítems pueden ser modificados por las persona(s) encargadas de la configuración del sistema.

•      Descripción Regla: La descripción de regla es el nombre largo para identificar la regla de manera funcional o de negocio.

•      Nombre Corto: La descripción corta de regla para identificar la regla de manera funcional o de negocio.

•      Prioridad: Esta es la prioridad para aplicar la regla.

•      Acción Regla: Esta una lista de selección de acciones que se pueden aplicar a las regla.

o  NO validar: Este ítem permite marcar cuando la regla no se valida.

o  No continua (Detiene el proceso): Este ítem permite marcar que si la regla en caso de incumplirla detiene el ingreso de la orden.

o  Genera Advertencia: Este ítem permite marcar que si la regla en caso de incumplirla continúa y muestra la advertencia.

o  Requiere Justificación: Este ítem permite marcar cuando la regla requiere justificación.

o  Requiere Aprobación: Este ítem permite marcar cuando la regla requiere aprobación.

•      Causas Justificación: En este texto se permite adicionar la lista de justificaciones para la regla; cuando se configure que requiere justificación.

A continuación se ofrece un listado de las reglas que actualmente aplican por módulos en el sistema

Esta regla debe ser de carácter general pero a su vez debe permitir excepciones para algunos clientes los cuales gozan de aprobación para superar el monto.
