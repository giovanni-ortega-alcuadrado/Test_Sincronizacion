Encabezado

1.   Número de Orden: Identificador único de la orden de tesorería. Este el número de agrupación de todas las formas de pago que se incluyan en la orden.

2.   Estado de la orden. Este campo muestra los diferentes estado por lo que pasa la orden. Estos puede ser: Pendiente, Cumplida o Cancelada.

3.   Documento autorización pago: En este ítem se permite adjuntar un archivo (.pdf, .xls, .doc, etc) donde el cliente autoriza a la firma para realizar el pago. No es obligatorio.

4.   Receptor: Lista que permite seccionar el receptor(es) a los cuales tienen permiso el usuario que ingresa la orden de tesorería. Este campo es obligatorio para continuar ingresando la orden.

5.   Tipo producto: Lista de selección de los tipos de productos a los cuales el receptor tiene permisos. Este campo es obligatorio para continuar ingresando la orden.

6.   Fecha orden: Fecha de la orden, por defecto presenta la fecha actual. Solo se pueden ingresar órdenes iguales a la fecha actual o mayores. Campo de tipo obligatoria.

7.   Cliente (nombre, tipo identificación, nro. documento): En este campo se puede seleccionar el Cliente al cual corresponde la orden. Este campo es obligatorio para continuar ingresando la orden.

Ejemplo del buscador de cliente.

Si de da clic en el botón de la lupa sin digitar datos se trae todos los clientes disponibles del receptor con el tipo de producto seleccionado.

Si se desea realizar una búsqueda específica se ingresa el  código OYD del cliente completo o parte, numero de documento de cliente o parte, nombre completo o parte.

8.   Saldo actual. Este valor corresponde a la consulta de saldos del cliente a la fecha. Este saldo se puede ver desagregado con las siguientes opciones.

9.   Totales de la orden. Saldo desagregado por forma de pago por orden.
