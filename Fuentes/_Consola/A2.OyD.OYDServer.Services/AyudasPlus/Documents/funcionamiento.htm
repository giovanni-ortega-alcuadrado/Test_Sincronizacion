Funcionamiento

1.   Se creó la opción menú  OyD plus>>Ordenes>>Carga masiva órdenes.

2.   Una vez se da clic en esta opción se debe seleccionar el tipo negocio y el tipo operación de las ordenes que desea carga, luego le permitirá escoger un archivo plano de carga de órdenes.

Tipo Negocio. Para cada uno de los tipos de negocio existe una hoja con estructura  diferente en la plantilla de carga.  A continuación se muestra una imagen de la plantilla.

Ver campos disponibles por tipo de negocio en el ítem Lista campos disponibles.

Tipo operación. Este puede ser compra o venta.

3.   Las características de este archivo plano de carga son las siguientes.

a.   El número de columnas será variable; es decir puede tener el número de columnas que el usuario considere, con la restricción que los campos existan en orden y con un nombre determinado en una lista, estos campos corresponden a los campos que se ingresan en la orden y homologan con los datos de la tabla de Órdenes.

Ejemplo: strIDComitente, strIDEspecie, pdblCantidad

b.   Los campos pueden estar en cualquier orden.

Ejemplo: strIDEspecie, pdblCantidad, strIDComitente.

c.   El tipo de producto del cliente tampoco es opción para cargar en el plano debido a que se define es el código del cliente.

d.   Si en el archivo plano de carga solo se tiene el Id de cliente se debe colocar por defecto el ordenante líder y la primera cuenta deposito que se encuentre. En caso de tener  múltiples cliente se debe ignorar  los datos por defecto colocados en la pantalla.

e.   Si en el archivo plano de carga se tiene una especie de acciones se ignoran los datos faciales.

f.    El archivo puede tener el deposito DECEVAL O DCV y la cuenta del depósito del cliente en cuyo caso se validara que ese relacionada con el cliente. Si la cuenta no está en el plano en la pantalla se deberá seleccionar el depósito y el sistema buscara la primera cuenta asociada que tenga el cliente. El archivo también podrá tener solo la cuenta depósito, pero en la pantalla deberá seleccionar el depósito para que el sistema valide que pertenece a ella.

g.   Para el manejo de las de las especies que vengan en el plano es que si es ACCIONES no debe traer los faciales, si es renta fija debe traer los faciales si no los trae deben estar matriculados en la pantalla.

h.   Se define que el ordenante del cliente para cada orden es el líder del cliente.

i.    Lo mismo que el receptor y el 100% de la comisión para el receptor líder del cliente.

j.    Al cargar los datos del plano que dependan de una lista o de una tabla relacionada, en el plano deberán estar son los códigos de retorno de la lista o el código de la tabla relacionada. Ejemplo: T = Transferencia en la forma de pago, el archivo tendrá el valor T y el proceso validara que la T si sea un dato real en la aplicación. Otro ejemplo es para el cliente, el campo en el plano es el código del cliente y el sistema lo validara que este activo en la tabla de clientes y que no sea ordenante.

Que puede suceder cuando se da clic al cargar o guardar órdenes.

k.   El proceso deberá validar que la combinación de datos del plano con los datos de la pantalla permitan crear una orden y no queden pendientes datos por definir en una captura manual de la aplicación.

l.    Luego que pase las validaciones básicas de las órdenes, se procede a ejecutar las reglas.

•      Si se presenta una orden con una regla que “No continua (Detiene el proceso)”,  Esa orden no es cargada y se procesan las demás.

•      Si se presenta una orden con una regla que “Requiere Justificación” o “Requiere Aprobación” con aprobar una o justificar todas los órdenes se aceptan y continúa con el proceso de todas las órdenes que incumplieron estas reglas. Nota las órdenes que requieren aprobación se van a autorizaciones tal cual como funciono actualmente.

•      Si se presenta una orden con una regla que “Genera Advertencia” se acepta la validación y no se vuelve a presentar para las demás órdenes que presenten la misma regla.

m.  Luego. se da clic en el botón “Confirmar y finalizar”.

n.   Se da clic en SI para aprobar.

o.   Se da clic en SI.

p.   Cuando termina el proceso se presenta una pantalla con los números de ordenes generado exitosamente, y una pantalla de inconsistencia o reglas que incumplen.
